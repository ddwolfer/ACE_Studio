# 前端設計需求書 — ACE-Step 遊戲配樂生成器

> 用途：餵給 Pencil 生成設計稿 → 之後依此實作前端。
> 平台：**獨立網頁 (HTML / React)**，串接 ACE-Step 的 `acestep-api` 後端。
> 視覺：**音樂工作站深色風 (DAW-like)** — 暗底、面板分區、波形、推桿質感。

---

## 1. 一句話定位

「一個像 Ableton/FL Studio 那樣的深色介面，但只做一件事：**用文字快速產生遊戲 BGM 與音效**。」
目標使用者：獨立遊戲開發者 / 個人創作者。核心訴求：**3 步出歌、不被一堆專有名詞淹沒**。

### 🟦 B. 左側 — 遊戲場景範本快選 (Preset Library)

## 9. Prompt 合成模型（重要架構決策｜已簡化為兩段）

最終送進模型的 caption 由**兩段串接**而成：

```
最終 prompt(caption) = 基底 prompt(base) + 額外 prompt(extra)
```

| 概念 | 定義 | 來源 | 角色 |
|------|------|------|------|
| **基底 prompt (Base)** | 一段可重用的完整風格描述 | 場景範本 / 我的模板 / 直接打字 | 主體 base |
| **額外 prompt (Extra)** | 這次臨時追加的 tag（批次時=套用到所有場景的共用風格） | 各頁的「額外 prompt」欄 | 後綴 extra |

- **重要決策（已採納）**：原本的「模板前綴」與「場景基底」**合併為單一「基底」**。理由：兩者本質都是「可重用的 prompt 文字」，而 ACE-Step 的 tags 是逗號分隔的控制信號、順序不敏感，無需分兩格。
- **連帶結論**：「場景範本」與「模板」是**同一種物件**＝「存好的基底 prompt + 可選預設參數」，差別只在內建 vs 使用者自存。左欄即兩組同類卡片（場景範本 / 我的模板），批次的「選擇場景」清單也同時包含使用者模板。
- 兩段任一可為空。單首生成頁的 caption 欄本身就能自由打字（等同直接編輯 base，再用進階區/額外欄補 extra）。
- **批次頁面**：勾選多個基底（場景/模板）+ 一段共用的額外 prompt；每個排入佇列的項目 = `該基底 + extra`，並提供「即時合成預覽」顯示實際送出字串。

## 10. ⚠️ 實作契約：每首歌都要記錄完整 prompt

因為底部播放列有「**複製 prompt**」按鈕、右側音檔庫有「重新生成 / 存為模板」，所以**每一次生成都必須把「最終送出的完整資訊」與音檔一起持久化**。需記錄的欄位：

| 欄位 | 說明 |
|------|------|
| `final_caption` | 兩段合成後實際送進模型的完整 caption |
| `base` / `extra` | 合成前的兩段原始字串（供「重新編輯」還原） |
| `lyrics` | 歌詞（或 `[Instrumental]`） |
| `params` | model、duration、steps、cfg、seed、bpm、key、infer_method、輸出格式 |
| `template_id` | 若套用了模板，記下是哪個 |
| `scene` | 若來自場景範本，記下場景名（供音檔庫篩選） |
| `audio_path` | 產出的音檔路徑（「打開本地目錄」用） |
| `created_at` | 生成時間 |

建議實作：與音檔同目錄放一個 `<檔名>.json` sidecar，或一個集中式 `library.json` 索引。
- 「複製 prompt」→ 複製 `final_caption`（或含 params 的完整描述）。
- 「打開本地目錄」→ 用 `audio_path` 開啟所在資料夾（Windows：`explorer /select,<path>`）。
- 「重新生成」→ 讀回 base/extra/params，換新 seed 再跑。
- 音檔庫「存為模板」→ 把 base（+ 預設參數）寫成一個模板。

> 對應後端：`scripts/generate.py` 之後要在輸出時一併寫出上述 sidecar；ComfyUI 路線則靠 `SaveAudio` 的 metadata 或額外節點記錄。

## 11. 設定頁 (Settings)

原本規劃為 ⚙ 抽屜（第 4-F 節），設計稿已實作為**置中 modal 面板**（由頂部 ⚙ 開啟）：

## 12. AI 助手（Claude 聊天室）

右欄改為分頁 **「AI 助手 / 音檔庫」**，預設顯示 AI 助手。讓使用者用自然語言描述需求，Claude 直接幫忙設定與生成。

**UI（已實作於設計稿）**
- 分頁切換：`AI 助手` / `音檔庫`。
- 對話區：助手訊息（左、灰泡）、使用者訊息（右、青綠泡）。
- **動作卡（Action Card）**：Claude 的建議以結構化卡片呈現，附按鈕：
  - 單首建議 → 顯示建議的 caption + 長度 + `套用並生成`。
  - 整套規劃 → 顯示場景清單 + `全部排入批次`。
- 底部輸入列：文字輸入 + 送出鈕。

**實作方式（build 階段）**
- 串 **Claude API（Anthropic）** 的 **tool-use**：定義工具如 `fill_single_form`、`queue_batch`、`save_template`、`set_params`；Claude 解析對話後回傳工具呼叫，前端執行（填表單 / 排佇列 / 套模板）。
- **與 ACE-Step 5Hz LM 的分工**：Claude＝對話層（理解需求、規劃整套配樂、寫/改 prompt、解釋）；5Hz LM＝模型內部把最終 prompt 展開成音樂。兩者互補不衝突。
- 對話與動作建議：每個「套用並生成」最終仍走第 10 節的記錄契約（保存 base/extra/params）。
- 需要 Anthropic API key（放後端，勿暴露於前端）。實作時參考 `claude-api` 指南挑模型與 tool-use 格式。
